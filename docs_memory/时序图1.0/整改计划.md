# YesBoss记忆模块接口与时序图整改计划

## 📊 审查结果总结

### 核心问题
- **未定义接口**：7个新接口（TriggerService、CaptionGenerator、SummaryGenerator、EmbeddingService、LlmClient扩展、MemoryExtractor、ChatMessageRepository扩展）
- **方法名错误**：findBySimilarity → searchBySimilarity
- **消息格式不规范**：未严格遵循"接口名::方法名"格式

### 工作量评估
- **总工作量**：约13人天
- **最复杂**：CaptionGenerator（3人天，需集成多模态API）
- **最简单**：LlmClient扩展（0.5人天）

## 📋 Phase 1：核心接口定义（2-3天）

### 1.1 新增接口清单

#### TriggerService（触发条件检查服务）
```java
package tech.yesboss.memory.trigger;

public interface TriggerService {
    /**
     * 检查是否满足触发条件
     */
    TriggerResult checkTriggerConditions();

    /**
     * 检查时间间隔条件
     */
    boolean checkIntervalCondition();

    /**
     * 检查对话数量条件
     */
    boolean checkEpochMaxCondition();

    /**
     * 获取未处理的对话
     */
    List<ChatMessage> getUnprocessedMessages(long intervalMs, int epochMax);

    /**
     * 标记对话为已处理
     */
    void markMessagesAsProcessed(List<String> messageIds);
}

public class TriggerResult {
    private boolean shouldTrigger;
    private TriggerReason reason;  // INTERVAL, EPOCH_MAX
    private List<ChatMessage> messages;
}
```

## 🛠️ Phase 2：服务层实现（2-3周）

### 2.1 实施优先级（基于产品评估）

#### 第一优先级（MVP必须）
1. **TriggerService**（1.5人天）
   - 定时任务检查
   - 条件判断逻辑
   - 对话获取和标记

2. **ChatMessageRepository扩展**（1人天）
   - 未处理对话查询
   - 批量标记接口

3. **MemoryService核心**（2人天）
   - 基础记忆提取流程
   - 简化的RAG检索

#### 第二优先级（MVP后1个月）
4. **EmbeddingService**（2人天）
   - 基础向量化
   - 批处理支持

5. **ContentProcessor**（2.5人天）
   - 核心摘要生成
   - 简化的prompt模板

#### 第三优先级（后续迭代）
6. **ContentProcessor完整版**（+1.5人天）
   - 多模态支持
   - 外部API集成

### 2.2 技术选型

#### Embedding模型
- **默认**：text-embedding-3-small（性价比高）
- **文档**：text-embedding-3-large（高精度）
- **成本**：$0.0001/1K tokens

#### LLM模型
- **摘要生成**：gpt-3.5-turbo（成本效益平衡）
- **RAG判断**：gpt-4o-mini（快速且准确）

#### 多模态API
- **图像**：OpenAI Vision API
- **音频**：OpenAI Whisper
- **视频**：FFmpeg提取帧 + Vision API

### 3.2 消息格式规范

#### 统一格式
```markdown
participant 服务名 as 显示名称
服务名->>接口名::方法名(参数)
接口名-->>服务名: 返回值
```

#### 示例
```markdown
participant TriggerSvc as TriggerService
TriggerSvc->>ChatMsgRepo::getUnprocessedMessages(intervalMs, epochMax)
ChatMsgRepo-->>TriggerSvc: List<ChatMessage>
```

## 🎯 MVP范围建议

### 功能范围
```
✅ 第一版MVP（3-4周）
├── 自动触发（interval/epoch_max）
├── Resource生成（仅conversation模态）
├── Snippet提取（仅profile/event/knowledge）
└── 简单RAG（两层：Preference + Snippet）

🚀 第二版（+2-3周）
├── 多模态支持（document/image）
├── 完整Snippet提取（6种类型）
├── 三层RAG（加入Resource层）
└── Preference自动更新

💡 第三版（+2周）
├── 所有模态支持
├── 批量处理优化
└── 高级特性（记忆去重、合并）
```

### 成本预估
- **LLM调用成本**：$10-20/天（基于使用量）
- **向量存储**：SQLite（免费）或 PostgreSQL+$5/月
- **总月成本**：约$300-600
