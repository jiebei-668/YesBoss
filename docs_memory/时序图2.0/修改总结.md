## 修改概述

根据审查意见，对时序图2.0版本进行了全面修正，主要解决了以下问题：

## 接口文档更新

### ContentProcessor接口补充

在 `记忆持久化模块v2.0.md` 的ContentProcessor接口中添加了以下方法：

```java
/**
 * 批量生成对话描述
 * @param contents 对话内容列表
 * @return 对话描述列表
 */
List<String> batchGenerateCaptions(List<String> contents);

/**
 * 从Snippets中提取Preference名称
 * @param snippets Snippet列表
 * @return Preference名称
 */
String generatePreferenceName(List<Snippet> snippets);

/**
 * 生成初始Preference摘要
 * @param snippets Snippet列表
 * @return 初始Preference摘要
 */
String generateInitialPreferenceSummary(List<Snippet> snippets);
```

## 统一规范

### Participant命名规范
- ✅ Repository一律使用全称：ResourceRepository, SnippetRepository, PreferenceRepository, ChatMessageRepository
- ✅ Service使用全称：TriggerService, ContentProcessor, EmbeddingService, MemoryService, MemoryManager
- ❌ 不使用简称：Repo, Service等
- ❌ 不包含外部服务：LLM等

### 消息格式规范
- ✅ 标准格式：`[接口名]::[方法名(参数)]`
- ✅ 示例：
  - `TriggerService::checkTriggerConditions()`
  - `ContentProcessor::generateCaption(conversationContent)`
  - `EmbeddingService::generateEmbedding(text)`
  - `ChatMessageRepository::markMessagesAsProcessed(messageIds)`

### 方法名规范
- ✅ 使用接口文档中定义的确切方法名
- ✅ 参数名使用通用名称（text, contents等）而非具体名称（summary, caption等）
