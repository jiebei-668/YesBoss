# 缓存机制实现文档

## 概述

YesBoss内存持久化模块实现了一个高性能的多级缓存系统，旨在减少数据库访问、提高响应速度并降低系统负载。

## 架构设计

### 多级缓存架构

```
┌─────────────────────────────────────────────────┐
│           应用层 (MemoryService)                 │
└───────────────────┬─────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────┐
│          CacheManager (缓存管理器)               │
└───────────────────┬─────────────────────────────┘
                    │
        ┌───────────┴───────────┐
        │                       │
┌───────▼────────┐    ┌────────▼────────┐
│  L1 Cache      │    │  L2 Cache       │
│  (In-Memory)   │    │  (Optional)     │
│  - Fast        │    │  - Slower       │
│  - Small       │    │  - Larger       │
│  - Primary     │    │  - Backup       │
└────────────────┘    └─────────────────┘
```

### 核心组件

1. **Cache接口**: 定义缓存操作的标准接口
2. **InMemoryCache**: 基于内存的L1缓存实现
3. **MultiLevelCache**: 多级缓存实现，支持L1和L2
4. **CacheManager**: 缓存管理器，负责创建和管理缓存实例
5. **CacheConfig**: 缓存配置类
6. **CacheStatistics**: 缓存统计信息
7. **CacheMetrics**: 监控指标收集

## 主要功能

### 1. 基本缓存操作

```java
// 获取缓存
Cache<String, Object> cache = CacheManager.getDefault().getCache("myCache");

// 存储数据
cache.put("key", value);

// 获取数据
Optional<Object> value = cache.get("key");

// 删除数据
cache.evict("key");

// 清空缓存
cache.clear();
```

### 2. 带过期时间的缓存

```java
// 存储带过期时间的数据
cache.put("key", value, Duration.ofHours(2));
```

### 3. 自动加载（Cache Loader）

```java
// 如果缓存中不存在，自动加载
Object value = cache.getOrCompute("key", key -> {
    // 从数据库加载
    return database.load(key);
});
```

### 4. 异步操作

```java
// 异步获取
CompletableFuture<Optional<Object>> future = cache.getAsync("key");

// 异步存储
CompletableFuture<Void> future = cache.putAsync("key", value);
```

### 5. 缓存统计

```java
// 获取统计信息
CacheStatistics stats = cache.getStatistics();

long hits = stats.getHitCount();
long misses = stats.getMissCount();
double hitRate = stats.getHitRate();
double avgLoadTime = stats.getAverageLoadTimeMillis();
```

### 6. 多级缓存

```java
// 创建多级缓存
CacheConfig l1Config = CacheConfig.builder()
    .maximumSize(10000)
    .expireAfterWrite(Duration.ofHours(1))
    .build();

CacheConfig l2Config = CacheConfig.builder()
    .maximumSize(100000)
    .expireAfterWrite(Duration.ofHours(24))
    .build();

MultiLevelCache<String, Object> cache =
    new MultiLevelCache<>(l1Config, l2Config);
```

## 配置说明

### application-memory.properties

```properties
# 基础配置
cache.maxSize=10000                    # 最大缓存条目数
cache.expireAfterWrite=1h              # 写入后过期时间
cache.expireAfterAccess=30m            # 访问后过期时间
cache.recordStats=true                 # 启用统计
cache.concurrencyLevel=4               # 并发级别

# 多级缓存配置
cache.multiLevel.enabled=false         # 是否启用L2缓存
cache.multiLevel.l2.maxSize=100000     # L2缓存大小
cache.multiLevel.l2.expireAfterWrite=24h  # L2过期时间

# 监控配置
cache.metrics.slowOperationThreshold=100  # 慢操作阈值(ms)
cache.metrics.hitRateAlertThreshold=0.8  # 命中率告警阈值
```

### 环境变量覆盖

任何配置都可以通过环境变量覆盖（前缀：`YESBOSS_CACHE_`）：

```bash
export YESBOSS_CACHE_MAXSIZE=20000
export YESBOSS_CACHE_EXPIREAFTERWRITE=2h
export YESBOSS_CACHE_MULTILEVEL_ENABLED=true
```

## 性能特性

### 1. 线程安全

- 使用`ReadWriteLock`实现读写分离
- 使用`ConcurrentHashMap`存储数据
- 所有操作都是线程安全的

### 2. LRU淘汰策略

- 当达到最大容量时，自动淘汰最少使用的条目
- 基于访问队列实现高效的LRU

### 3. 时间过期

- 支持写入后过期（expireAfterWrite）
- 支持访问后过期（expireAfterAccess）
- 定期清理过期条目

### 4. 批量操作优化

- 支持批量put/get操作
- 减少锁竞争

### 5. 异步操作

- 所有操作都有异步版本
- 使用专用线程池处理异步任务

## 监控指标

### 统计指标

- **命中率**: 缓存命中次数 / 总请求数
- **请求次数**: 读、写、删除操作计数
- **平均加载时间**: 从缓存加载数据的平均时间
- **淘汰次数**: 因容量限制被淘汰的条目数

### 性能指标

- **响应时间**: 操作的平均响应时间
- **吞吐量**: 每秒处理的操作数
- **慢操作数**: 超过阈值的操作数

### 告警机制

- **低命中率告警**: 命中率低于阈值时触发
- **高容量告警**: 缓存大小接近上限时触发
- **慢操作告警**: 慢操作比例过高时触发

## 使用示例

### 1. 资源缓存

```java
// 创建资源缓存
Cache<String, Resource> resourceCache = CacheManager.getDefault()
    .getCache("resources");

// 缓存资源
resourceCache.put(resourceId, resource);

// 获取资源
Optional<Resource> cached = resourceCache.get(resourceId);
if (cached.isPresent()) {
    return cached.get();
}

// 从数据库加载
Resource resource = resourceRepository.findById(resourceId);
resourceCache.put(resourceId, resource);
return resource;
```

### 2. 片段缓存

```java
// 自动加载模式
Snippet snippet = snippetCache.getOrCompute(snippetId, id -> {
    return snippetRepository.findById(id);
});
```

### 3. 批量预热

```java
// 预热缓存
List<Resource> resources = resourceRepository.findAll();
resources.forEach(resource -> {
    resourceCache.put(resource.getId(), resource);
});
```

### 4. 监控和告警

```java
// 获取指标
CacheMetrics metrics = cacheManager.getMetrics("resources");
CacheMetrics.CacheMetricsSummary summary = metrics.getSummary();

// 检查告警
List<Alert> alerts = metrics.getActiveAlerts();
alerts.forEach(alert -> {
    logger.warn("Alert: {} - {}", alert.getId(), alert.getMessage());
});
```

## 最佳实践

### 1. 合理设置缓存大小

```java
// 根据内存容量和访问模式设置
CacheConfig config = CacheConfig.builder()
    .maximumSize(calculateOptimalSize())  // 基于实际需求计算
    .build();
```

### 2. 设置适当的过期时间

```java
// 热数据：短过期时间
cache.put("hotData", value, Duration.ofMinutes(5));

// 冷数据：长过期时间
cache.put("coldData", value, Duration.ofHours(24));
```

### 3. 使用缓存加载器

```java
// 避免缓存穿透
Value value = cache.getOrCompute(key, k -> loadFromDatabase(k));
```

### 4. 监控缓存效果

```java
// 定期检查缓存命中率
CacheStatistics stats = cache.getStatistics();
if (stats.getHitRate() < 0.8) {
    // 调整缓存策略
    logger.warn("Low cache hit rate: {}", stats.getHitRate());
}
```

### 5. 及时清理

```java
// 不再需要时清理缓存
cacheManager.removeCache("oldCache");
```

## 性能基准

### 预期性能指标

- **读操作（命中）**: < 1ms
- **读操作（未命中）**: < 100ms（包含加载时间）
- **写操作**: < 1ms
- **删除操作**: < 1ms
- **命中率**: > 80%
- **吞吐量**: > 10,000 ops/s

### 内存使用

- 每个缓存条目约 1KB（包含对象引用）
- 10,000条目约 10MB
- 100,000条目约 100MB

## 故障排查

### 问题1: 命中率低

**可能原因**:
- 缓存容量太小
- 过期时间太短
- 数据访问模式不符合LRU

**解决方案**:
- 增加缓存大小
- 延长过期时间
- 分析访问模式

### 问题2: 内存占用过高

**可能原因**:
- 缓存大小设置过大
- 对象过大

**解决方案**:
- 减少缓存大小
- 优化缓存对象大小

### 问题3: 慢操作过多

**可能原因**:
- 缓存加载慢
- 锁竞争严重

**解决方案**:
- 优化数据加载逻辑
- 增加并发级别
